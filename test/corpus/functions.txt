Forward function declaration — not supported, produces ERROR
================================================================================

function foo;
function foo() { return 1; }

--------------------------------------------------------------------------------

; `function foo;` is not valid ucode. The grammar produces an ERROR node
; and the semicolon is parsed as an empty_statement.

(program
  (ERROR
    (identifier))
  (empty_statement)
  (function_declaration
    name: (identifier)
    parameters: (formal_parameters)
    body: (statement_block
      (return_statement
        (number)))))
